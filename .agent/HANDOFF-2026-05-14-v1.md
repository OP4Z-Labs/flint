## What Flint v1.0 ships

Eleven commands, three template variants, and a stable programmatic API
exposing the same surface for Node-scripting consumption. All commands
support `--json` for CI scripting.

| Command | One-liner |
| --- | --- |
| `flint init` | Scaffold Cloudflare Pages config into an existing Vite + React + TS repo |
| `flint create-app <name>` | Bootstrap a fresh Vite + React + TS app with Pages wiring |
| `flint deploy` | Build + pre-flight + `wrangler pages deploy`, with health-ping |
| `flint upgrade` | Drift detection + 3-way merge against current templates |
| `flint doctor` | Full-stack diagnostics (node / pm / wrangler / auth / repo) |
| `flint uninstall` | Manifest-aware deletion of Flint-scaffolded files |
| `flint configure` | Walk through provisioning every Cloudflare resource in `wrangler.toml` |
| `flint add {kv,r2,secret,pwa,auth,rate-limit}` | Six additive scaffolds |
| `flint auth {init,status,doctor,rotate,purge}` | Cloudflare API token lifecycle |
| `flint telemetry {show,purge,export}` | Local telemetry log management |
| `flint config` | Global preferences (telemetry on/off) |

Three template variants: `static-spa`, `pages-functions`, `pages-fullstack`.
Verified against three real apps: Portfolio, Chorus, Blaze.

## Manifest schema v1: LOCKED

The `flint.manifest.json` schema is at version 1 and is frozen for the
1.x line. Any change to the manifest shape — adding fields to entries,
renaming history fields, changing the hash algorithm — requires a 2.0
release.

Schema lives in `src/util/manifest.ts`. Reading: `readManifest()`. Writing:
`writeManifest()`. Classification: `classifyAll()` / `classifyFile()`.

## What's deferred to v1.x+

- Adopting `windows-latest` into CI matrix (doubles CI minutes).
- Auto-regenerating `start/commands.md` from `--help` output.
- Adoption-gap detection (suggesting `flint add ratelimit` when the
  Blaze-variant template includes it but the project doesn't have it).
- Custom domain attachment (`wrangler pages domain` wrapping).
- A Flint-hosted telemetry endpoint (currently local-log only; users
  must self-host a collector if they want to forward).
- `@op4z/edge-content` runtime extraction.
- Programmatic API integration tests on Node 20/22/24 matrix.
- Cadence audits running on the docs site content.
- CI-driven docs site deploy on tag.

None are blockers for 1.0.
